# Configure Project

Set up or update project-specific Cursor configuration.

## Ask me about:

### Project Basics
1. What's the project name and primary purpose?
2. Is this Next.js App Router or Pages Router?
3. TypeScript strict mode? (yes/no)
4. CSS approach? (Tailwind/CSS Modules/styled-components)
5. UI library? (shadcn/MUI/Chakra/none)

### Infrastructure
6. Vercel project name? (for deployments)
7. Production URL?
8. Staging URL? (if applicable)
9. Linear team name or ID?
10. Do you use Supabase? (project ID if yes)

### Development Patterns
11. API pattern? (REST/GraphQL/tRPC)
12. State management? (Context/Zustand/Redux/none)
13. Form handling? (react-hook-form/Formik/native)
14. Testing framework? (Vitest/Jest/none)
15. Monorepo? (yes/no)

### Workflow Preferences
16. Branch naming pattern?
17. Commit message style? (conventional/simple)
18. PR size preference? (small/medium/flexible)

## After answering:
1. Update .cursor/rules/project.md
2. Create project-specific commands
3. Update .env.template
4. Generate setup documentation

Ready to configure your project!
